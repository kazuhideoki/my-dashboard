# 開発ドキュメント

## タスクへの取り組み方

### 手順

**必ず以下の手順を守ること**
**【最重要】特に指示がない限り、１フェーズの作業完了ごとにユーザーのレビューを受けてから次のフェーズに進むこと**
**スコープ外の作業はしないこと**

1.  型やインターフェースの定義作成/変更をする。
    - **2に進む前に、必ずユーザーによるレビューをうけること**
2.  仕様としてテストケース表現を記述(it("~") を記述)し保存する
    - 既存テストで保証できる挙動があるか確認すること。できるなら新たにテストケースを追加する必要は無い。。
    - **3に進む前に、必ずユーザーによるレビューをうけること**
3.  テストの内容を作成(修正)する
    - 対象テストが `cargo test` を実行して **失敗することを確認**
    - **4に進む前に、必ずユーザーによるレビューをうけること**
4.  実装する。必ず以下のコマンドも実行する
    - `cargo check` を実行してエラーがないことを確認
    - `cargo fmt` を実行してエラーがないことを確認
    - `cargo clippy -- -D warnings` を実行してエラーがないことを確認
    - `cargo test` を実行して成功することを確認
    - ※ 失敗した部分を修正する
    - 最後に実行した報告をする

※途中で気になったことは、ユーザーに断った上で TODO コメントとして残すこと

### 参照ドキュメント

- タスク関連ドキュメント `plan*.md` を確認して作業すること。
- 新しいライブラリ/技術を利用するときは必ず検索してインターネット上の情報を参照すること

## 開発フェーズ

現在MVPリリースをしたものの、社内向けかつまだまだ開発中である。
多くの場合互換性は考慮しなくていい。必要な場合は説明し、許可を得ること
